Ce que font les 3 éléments

Chaque layer a un rôle précis. Ensemble, ils couvrent la totalité de la chaîne : ingestion de données brutes → analyse → décision → affichage interactif.

Layer 1 — Moteur

TRESORIS · Engine Python

13 moteurs de calcul déterministes et probabilistes. Le cerveau analytique. Zéro UI, zéro LLM nécessaire — pure logique financière.

  • 💳
    FinanceEngine
    Trésorerie, runway, prévisions cash, RiskItems — base de tout
  • 🔍
    ClientRiskScorer
    Score 0–100 par client (A/B/C/D), formule pondérée 4 axes
  • 📈
    SmartForecaster
    Prévision de paiement facture par facture avec intervalles de confiance
  • 🚨
    EarlyWarningDetector
    Signaux faibles — détecte la dégradation avant la crise (delay, churn, concentration)
  • ⚡
    ActionPrioritizer
    Priorisation impact×0.7 + ease×0.3 — P0 à P3 avec quick wins
  • 📅
    SeasonalityEngine
    Facteurs août (×1.3), décembre (×1.15), Pâques (×1.1)
  • 📊
    MarginAnalyzer V3
    Marge contributive réelle : COGS + coûts cachés (financement DSO, recouvrement, SAV)
  • 👻
    CostDriftAnalyzer V3
    Coûts fantômes, effet cliquet, inflation interne, doublons SaaS
  • 🔗
    CausalAnalyzer V3
    Répond au POURQUOI : décompose une baisse de marge en causes chiffrées (mix, inflation, volume)
  • 📉
    VarianceAnalyzer V3
    Écart budget/réel décomposé — effet prix, effet volume, effet mix
  • 🎲
    StressTester V3
    Monte Carlo 10K simulations, VaR, worst-case, reverse stress (quel choc pour casser ?)
  • ⚖️
    DecisionArbiter V3
    Recruter vs sous-traiter, acheter vs louer, NPV, IRR, payback — avec impact trésorerie
Stack : Python pandas numpy scipy · API FastAPI · Deploy Railway/Vercel

Agent autonome qui orchestre la chaîne complète : ingestion → KPIs → ML/IA → simulation → scoring. Zéro dépendance UI.

  • 🔄
    DataProcessor
    Normalise n'importe quel CSV/Excel en FinancialRecord[] — validation + tri chronologique
  • 📐
    KPIEngine
    CA, Marge, Cash Flow, DSO, BFR — calculs purs, confidence 1.0
  • 🧠
    AnalysisOrchestrator
    Séquence : AnomalyDetector → CashFlowPredictions → AdvancedPatterns → FinSightScore™
  • 🤖
    AnomalyDetector (ML)
    Z-Score sur transactions — détecte valeurs aberrantes sans LLM
  • 🔮
    CashFlowPredictions (GPT-4)
    Forecasting 3–6 mois + alertes prédictives de tensions trésorerie
  • 🔍
    PatternDetector (GPT-4)
    Comportements clients, saisonnalité cachée, tendances non linéaires
  • 🏆
    FinSight Score™
    Note 0–100 sur 4 piliers : Cash / Margin / Resilience / Risk + benchmark sectoriel
  • ⚙️
    SimulationEngine
    What-If : réduction charges, accélération paiements, hausse prix — impact immédiat sur KPIs
Stack : TypeScript Node.js OpenAI GPT-4 TensorFlow.js · Stateless

Point clé : Les 3 layers sont découplés. TRESORIS peut être exposé via son API FastAPI sans DASHIS ni FinSight. DASHIS peut tourner en CLI sans frontend. FinSight peut appeler TRESORIS directement depuis ses API routes Next.js. C'est ce découplage qui rend le stack revendable en OEM, en API, en white-label ou en SaaS autonome.

Les 5 embranchements possibles

Le même stack peut prendre 5 formes commerciales différentes selon le partenaire, le timing et l'effort de dev. Ce ne sont pas des pivots — ce sont des couches parallèles activables à la demande.

🏢
SaaS PME Direct
FinSight comme produit B2B autonome. Le dirigeant uploade son export comptable, reçoit le score + rapport.
  • Abonnement mensuel ou mission one-shot (Diagnostic 2 490€)
  • Upsell vers Audit (6 990€) ou Decision System (12 500€)
  • Lead magnet → nurture email → funnel déjà en place
  • Cible : dirigeants PME 2–20 M€ sans DAF interne
Statut : Live sur finsight.zineinsight.com
⚙️
API OEM / Moteur Embarqué
TRESORIS vendu comme CPU financier intégré dans l'offre d'un partenaire. Le partenaire garde 65–75%.
  • Revenue share 25–35% ZineInsight / 65–75% partenaire
  • Upfront 10–30k€ + % sur chaque client activé
  • White-label possible : aucune mention ZineInsight
  • Exclusivité sectorielle / géographique négociable
Statut : Deal FINAVIA signé — 30/70, 10k€ upfront, France Finance PME 24 mois
🏦
Finance-as-a-Service (FaaS)
Le stack complet exposé via API RESTful stateless. Partenaires appellent /v1/score, /v1/anomalies, /v1/simulate.
  • Pricing pay-per-call : 0,10–0,50€/appel selon endpoint
  • Tier SLA : sandbox (gratuit) → growth → enterprise
  • Widget React <FinSightWidget /> embeddable 1 ligne
  • Webhooks : alerte franchissement DSO, runway critique
Endpoints clés : /v1/score · /v1/anomalies · /v1/simulate · /v1/forecast
🔌
Plug-in Qonto / Pennylane / Agicap
TRESORIS comme module de scoring et d'analyse embarqué dans les plateformes fintech existantes.
  • S'intègre dans leur flow : données bancaires → score FinSight™
  • Qonto : alertes trésorerie intelligentes dans le dashboard
  • Pennylane : scoring client + early warning dans le CRM compta
  • Agicap : causal analyzer + variance analyzer en overlay
Avantage : Ils ont la distribution, on a l'IP analytique qu'ils n'ont pas
🧑‍💼
DAF-as-a-Service (Cabinets)
Le stack vendu à des cabinets d'experts-comptables ou DAF externalisés. Chaque cabinet sert ses clients avec le moteur.
  • Multi-tenant déjà implémenté dans TRESORIS (Google Sheets poller)
  • 1 instance → N clients du cabinet
  • Dashboard white-label + rapports PDF brandés cabinet
  • Facturation cabinet : forfait mensuel par client actif
Modèle : 200–500€/mois par cabinet × 5–20 clients actifs chacun
Règle des embranchements : les branches 1 (SaaS direct) et 2 (OEM) sont actives maintenant. La branche 3 (FaaS API) nécessite 4–6 semaines de dev (versioning, auth, rate limiting). La branche 4 (plug-in Qonto/Pennylane) nécessite un accord partenariat — mais le code est déjà compatible. La branche 5 (cabinets) est activable aujourd'hui avec TRESORIS multi-tenant existant.
