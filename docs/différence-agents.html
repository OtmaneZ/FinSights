Cartographie de la Propriété Intellectuelle (IP)

Document de cadrage interne : Ligne de démarcation technologique et commerciale entre les actifs ZineInsight (SASU) et les projets de collaboration externe.

100% IP ZineInsight

Stack "SCORIS" (FinSight)

Nature : Infrastructure logicielle de notation financière, de prédiction de flux et de modélisation de risques.

  • Architecture : 3 Layers (TRESORIS / DASHIS / FinSight).
  • Finalité : Évaluation externe (Scoring 0-100), détection de signaux faibles, tests de résistance (Monte Carlo).
  • Logique Temporelle : Prédictive (Forecasting, Probabilités).
  • Data Input : Flux bancaires (JSON, API), FEC normés, données PME non structurées.
  • Modèle de vente : API White-Label, SaaS autonome, OEM intégrable (Agnostique).
Projet Collaboratif (4 DAFs)

Agent Contrôle de Gestion

Nature : Outil de reporting interne sur-mesure pour direction financière multi-entités.

  • Architecture : Agent dédié aux règles de gestion spécifiques d'un client.
  • Finalité : Gouvernance interne, analyse d'écarts (Budget vs Réel), reporting Comex.
  • Logique Temporelle : Rétrospective (Contrôle d'exécution budgétaire).
  • Data Input : Comptabilité analytique, centres de coûts, exports ERP lourds et structurés.
  • Modèle de vente : Outil livré dans le cadre d'une prestation de conseil (Sur-mesure / Custom).

🔒 Directive d'Ingénierie (Strict Boundary) : Aucun algorithme probabiliste (TRESORIS), aucun modèle de machine learning de détection d'anomalies (DASHIS), et aucun calcul des 4 piliers du Score FinSight™ ne doit être copié, instancié ou importé dans le repository de l'Agent Contrôle de Gestion. Les deux bases de code doivent rester strictement isolées pour protéger la valorisation de la SASU ZineInsight.
